Git & GitHub Handy Note

A quick reference guide for all essential Git commands

  • git --version – Check if Git is installed.
  • git status – Check the status of the repository.
  • git log – View commit history.
  • git log --oneline – View concise commit history.
  • git log --graph – View commit history with a branch graph.
  • git log -p – View commit history with changes.
  • git log --author="username" – View commits by a specific author.
  • git log --since="2023-01-01" --until="2023-12-31" – View commits within a date range.
  • git show <commit-hash> – Show details of a specific commit.
  • git diff – Show unstaged changes.
  • git diff --staged – Show staged changes.
  • git blame <file> – Show who last modified each line of a file.
  • git remote -v – List remote repositories.
  • git branch – List local branches.
  • git branch -r – List remote branches.
  • git branch -a – List all branches (local and remote).
  • git tag – List all tags.
  • git config --list – List all Git configurations.
  • git config --get user.name – Get the configured username.
  • git config --get user.email – Get the configured email.
  • git reflog – Show a log of all reference changes.

  • git init – Initialize a new repository.
  • git clone <repo-url> – Clone a repository.
  • git clone --branch <branch-name> <repo-url> – Clone a specific branch.
  • git clone --depth 1 <repo-url> – Clone only the latest commit.
  • git branch <branch-name> – Create a new branch.
  • git checkout -b <branch-name> – Create and switch to a new branch.
  • git switch -c <branch-name> – Create and switch to a new branch (Git 2.23+).
  • git tag <tag-name> – Create a lightweight tag.
  • git tag -a <tag-name> -m "message" – Create an annotated tag.
  • git add <file> – Stage a file for commit.
  • git add . – Stage all changes.
  • git commit -m "message" – Commit changes with a message.
  • git commit -am "message" – Stage and commit all tracked files.
  • git commit --amend – Amend the previous commit.
  • git commit -S -m "message" – Create a signed commit.

  • git checkout <branch-name> – Switch to an existing branch.
  • git switch <branch-name> – Switch to an existing branch (Git 2.23+).
  • git merge <branch-name> – Merge a branch into the current branch.
  • git rebase <branch-name> – Rebase the current branch onto another branch.
  • git rebase --interactive <commit-hash> – Interactive rebase.
  • git branch -m <new-branch-name> – Rename the current branch.
  • git branch -m <old-name> <new-name> – Rename a branch.
  • git reset --soft <commit-hash> – Reset to a commit, keeping changes staged.
  • git reset --mixed <commit-hash> – Reset to a commit, unstaging changes.
  • git reset --hard <commit-hash> – Reset to a commit, discarding all changes.
  • git revert <commit-hash> – Create a new commit that undoes a previous commit.
  • git cherry-pick <commit-hash> – Apply a specific commit to the current branch.
  • git remote add <name> <repo-url> – Add a new remote repository.
  • git remote set-url <name> <new-url> – Change the URL of a remote repository.
  • git remote rename <old-name> <new-name> – Rename a remote repository.
  • git fetch <remote> – Fetch changes from a remote repository.
  • git pull <remote> <branch> – Fetch and merge changes from a remote branch.
  • git push <remote> <branch> – Push changes to a remote branch.
  • git push --set-upstream <remote> <branch> – Push and set the upstream branch.
  • git push --force – Force push changes (use with caution).
  • git push --force-with-lease – Force push changes only if the remote branch hasn't changed.
