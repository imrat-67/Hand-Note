Git and GitHub Handy Notes

A collection of useful Git and GitHub commands for developers.

1. Checking (Inspecting and Viewing)

Repository Status

git status

Check the status of the working directory (untracked, modified, staged files).

Commit History

git log

View commit history.

git log --oneline

View concise commit history (one line per commit).

git log --graph

View commit history with a branch graph.

git log -p

View commit history with changes (patch format).

git log --author="username"

View commits by a specific author.

git log --since="2023-01-01" --until="2023-12-31"

View commits within a date range.

git show <commit-hash>

Show details of a specific commit.

git diff

Show unstaged changes in the working directory.

git diff --staged

Show changes staged for the next commit.

git diff <commit1> <commit2>

Show differences between two commits.

git blame <file>

Show who last modified each line of a file.

git remote -v

List remote repositories (with URLs).

git branch

List local branches.

git branch -r

List remote branches.

git branch -a

List all branches (local and remote).

git tag

List all tags.

git config --list

List all Git configurations.

git config --get user.name

Get the configured username.

git config --get user.email

Get the configured email.

git reflog

Show a log of all reference changes (e.g., branch switches, resets).

2. Creating

Repositories

git init

Initialize a new Git repository in the current directory.

git clone <repository-url>

Clone a remote repository to your local machine.

git clone --branch <branch-name> <repository-url>

Clone a specific branch of a repository.

git clone --depth 1 <repository-url>

Clone only the latest commit (shallow clone).

Branches

git branch <branch-name>

Create a new branch.

git checkout -b <branch-name>

Create and switch to a new branch.

git switch -c <branch-name>

Create and switch to a new branch (Git 2.23+).

git tag <tag-name>

Create a lightweight tag.

git tag -a <tag-name> -m "tag message"

Create an annotated tag.

Commits

git add <file>

Stage a file for commit.

git add .

Stage all changes in the working directory.

git commit -m "commit message"

Commit staged changes with a message.

git commit -am "commit message"

Stage and commit all tracked files in one command.

git commit --amend

Amend the previous commit (e.g., change message or add files).

Keys and Tokens

ssh-keygen -t ed25519 -C "your_email@example.com"

Generate a new SSH key (Ed25519).

ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

Generate a new SSH key (RSA).

git config --global credential.helper store

Save HTTPS credentials permanently.

git config --global credential.helper cache

Cache HTTPS credentials for a short time.

gh auth login

Log in to GitHub CLI (GitHub Token).

gh auth status

Check GitHub CLI authentication status.

gpg --full-generate-key

Generate a new GPG key.

gpg --list-secret-keys --keyid-format LONG

List GPG keys.

git config --global user.signingkey <GPG-key-id>

Set a GPG key for signing commits.

git commit -S -m "signed commit"

Create a signed commit.
